Page Conditions Example

On the left side you can see some examples of page conditions. Pages can be:

  • Invisible
  • Readonly
  • Disabled

*The condition can depend on any type of control implemented in HybridForms (i.e. TextField, RadioBox, CheckBox, TextArea, ...)

Invisible Page

This input field can have any value and by entering text, the hidden page will become visable

Invisible and disabled pages are exclude in the PDF!

{ "cond": [ { "type": "field", "id": "hidden-page" } ] }
Disabled Page

To enable the page, you need to enter a specific input, like in this example "enable me"

{ "cond": [{ "type": "field", "id": "disabled-page", "val": "enable me" }], "else": "disabled" }
Readonly Page

To enable the page, you need to enter a specific input, like in this example "edit page"

{ "cond": [{ "type": "field", "id": "readonly-page", "val": "edit page" }], "else": "readonly" }
